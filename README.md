# 音声通訳（VoiceTrans / Android 単独動作）

スマートフォン単独（サーバ不要）で動作する音声通訳アプリ。Pixel 9a 以上を想定。

```
マイク(本体/USB) → 録音(押して話す・≤30秒) → Gemma 3n E4B(音声→原文＋翻訳) → 端末内TTSで発声
```

- **STT＋翻訳**：オンデバイス Gemma 3n E4B（音声入力対応モデル）。
- **TTS**：端末内蔵 TextToSpeech（オフライン可）。
- **2マイク**：本体マイク／USBマイクを個別指定。
- **言語**：言語1＝端末の設定言語（既定）、言語2＝選択した外国語。押したボタンの言語で認識。
- **UI**：言語1=右／言語2=左のチャット、文字サイズ、発声ON/OFF、印刷（全て/言語1/言語2）。
- UI文字・言語名は端末の言語に追従。**RAG なし**。

> 現時点では本リポジトリにアプリのソース／バイナリは含めていません。
> ここでは、本アプリが利用する **Gemma 3n E4B モデルの入手方法** を案内します。

---

## 翻訳モデル（Gemma 3n E4B）

- 形式：**`.litertlm`**（音声エンコーダ同梱・音声入力対応）。
  ※ `.task`（litert-preview）は音声エンコーダ非搭載のため音声入力には使えません。
- アプリ内のダウンロード機能で端末へ取得します（adb 不要）。保存先はアプリ専用領域のため、
  **アプリをアンインストールするとモデルも自動削除**されます。
- ダウンロード元はアプリ設定で選択可能：
  - **Web(HTTP)** … ホスト/ポート/ファイル名
  - **FTP** … ホスト/ポート/ファイル名
  - **Hugging Face** … URL＋アクセストークン
- 大容量（約5GB）のため、ダウンロードは **レジューム＋自動リトライ** 対応。WiFi 推奨。

### Hugging Face から入手する場合

Gemma はゲートモデルのため、ライセンス同意とアクセストークンが必要です。

1. https://huggingface.co/ にログイン。
2. モデルページで Gemma 利用規約に同意（HF もしくは Kaggle の同意画面で Accept）。
   - 音声対応版： `google/gemma-3n-E4B-it-litert-lm` の `gemma-3n-E4B-it-int4.litertlm`
3. fine-grained トークンを作成（https://huggingface.co/settings/tokens/new?tokenType=fineGrained）。
   - 権限は **「Read access to contents of all public gated repos you can access」** のみで可。
4. アプリ設定の「Hugging Face」モードに、上記モデルの resolve URL とトークンを設定してダウンロード。

> ⚠️ トークンは秘密情報です。本リポジトリ等に保存しないでください。

## 使い方

1. アプリ起動後、ヘッダ右の **「⬇ モデルDL」**（設定⚙️→翻訳モデルからも可）でモデルを取得。
   進捗％と予想残り時間を表示。完了でヘッダが「準備完了」になり、下のボタンが有効化。
2. ヘッダの言語2で翻訳先を選択。⚙️で設定（文字サイズ・マイク割当・言語・モデル）。
3. 下部の「言語1／言語2」ボタンを **押している間に話し、離す** と、文字起こし→翻訳→（発声ON時）読み上げ。
4. マイクが1本でも2本でも、押したボタンの言語で認識します。

## 動作要件・制約

- 対象：arm64・Android 12 (API 31) 以上（Pixel 9a 想定）。
- 音声は最長30秒のバッチ推論（長文は区切る）。
- 翻訳エンジン：MediaPipe LLM Inference（`com.google.mediapipe:tasks-genai`）。
- 初回のモデル読込・各推論のレイテンシは端末性能に依存。
